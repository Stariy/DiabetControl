# DiabetControl — учёт питания при диабете 1 типа

Программа для Windows, помогающая рассчитывать углеводы, хлебные единицы (ХЕ), гликемическую нагрузку (ГН) и дозы инсулина на основе съеденных продуктов и блюд. Создана специально для родителей детей с диабетом, но подойдёт всем, кто следит за своим рационом.

## Возможности

- **База продуктов** — хранение названий, КБЖУ на 100 г и гликемического индекса.
- **Редактор блюд** — создание собственных рецептов с указанием веса каждого ингредиента, а также кастрюли по умолчанию и типичного веса готового блюда.
- **База кастрюль** — список кастрюль с фото и весом (для точного вычета тары).
- **Калькулятор приёма пищи** — сбор блюд и отдельных продуктов в один приём, редактирование весов прямо в интерфейсе, автоматический пересчёт:
  - Калорий, белков, жиров, углеводов
  - Гликемической нагрузки (ГН)
  - Хлебных единиц (ХЕ) с настраиваемым коэффициентом (1 ХЕ = X г углеводов)
- **Сохранение истории** — каждый рассчитанный приём можно записать с датой, временем, дозой инсулина и примечанием. В любой момент можно просмотреть детали или скопировать состав обратно в калькулятор для повторного использования.
- **Наглядный интерфейс** — всё разбито по вкладкам, простое управление.

## Установка и запуск

### Вариант 1: Готовый .exe (рекомендуется)
1. Скачайте последний релиз из раздела [Releases](ссылка).
2. Распакуйте архив в любую папку.
3. Запустите `DiabetControl.exe`. База данных создастся автоматически, начальные продукты уже загружены.

### Вариант 2: Запуск из исходного кода (для разработчиков)
1. Установите Python 3.10 или выше.
2. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/yourusername/DiabetControl.git
   cd DiabetControl
   ```
3. (Рекомендуется) Создайте виртуальное окружение:
```bash
python -m venv venv
source venv/bin/activate  # для Linux/Mac
venv\Scripts\activate     # для Windows
```
4. Установите зависимости:

```bash
pip install Pillow
```
(другие библиотеки – только стандартные)

5. Запустите:

```bash
python main.py
```
## Структура проекта
- **main.py** — точка входа, главное окно с вкладками.
- **database.py** — работа с SQLite, создание таблиц, основные запросы.
- **products_tab.py** — вкладка управления продуктами.
- **pans_tab.py** — вкладка управления кастрюлями.
- **dishes_tab.py** — вкладка создания и редактирования блюд.
- **calculator_tab.py** — основная вкладка для расчёта приёма пищи.
- **history_tab.py** — вкладка истории с просмотром и копированием.
- **utils.py** — вспомогательные функции для расчётов (КБЖУ, ГН, ХЕ).
- **config.py** — настройки путей и параметров по умолчанию.
- **pans_photos/** — папка, куда сохраняются изображения кастрюль.

## Использование
### 1. Продукты
Откройте вкладку «Продукты».

Добавьте новый продукт (кнопка «Добавить») или отредактируйте существующий двойным щелчком.

Укажите название, калории, белки, жиры, углеводы (на 100 г) и гликемический индекс (по желанию).

### 2. Кастрюли
Во вкладке «Кастрюли» добавьте свои кастрюли, указав название, вес и фото (для наглядности).

### 3. Блюда
Перейдите на вкладку «Блюда».
Создайте новое блюдо («Новое»), введите название.

Выберите кастрюлю по умолчанию и укажите типичный вес готового блюда (эти данные будут подставляться в калькулятор).
Добавьте продукты в состав («Добавить продукт»), укажите вес каждого ингредиента в граммах.

Все изменения сохраняются автоматически.

### 4. Калькулятор
На вкладке «Калькулятор» собирайте приём пищи:

- **+ Блюдо** — выберите блюдо из списка. Появятся поля для кастрюли, веса готового блюда и веса порции, а также таблица с составом (веса продуктов можно менять прямо в таблице).

- **+ Продукт** — выберите отдельный продукт, укажите его вес.

При изменении любых параметров итоговые показатели (КБЖУ, ГН, ХЕ) пересчитываются мгновенно.

Если вы изменили веса продуктов в составе блюда и хотите сохранить новый рецепт, нажмите «Сохранить как рецепт».

По окончании нажмите «Записать приём», укажите дату/время, дозу инсулина и примечание. Приём сохранится в истории.

### 5. История
Во вкладке «История» отображаются все сохранённые приёмы.

Двойной щелчок по записи открывает подробный просмотр состава.

Кнопка «Копировать в калькулятор» загрузит выбранный приём обратно (все блюда будут развёрнуты в отдельные продукты с теми весами, которые были съедены). Это удобно для повторения похожих обедов.

## Настройки
Пока настройки задаются в файле config.py (коэффициент ХЕ = 12 г углеводов на 1 ХЕ). В будущем появится графический интерфейс для изменения.
